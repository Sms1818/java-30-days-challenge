# **Definition**

Java is a high-level, platform-independent, object-oriented programming language used to build scalable and maintainable web apps, mobile applications, and enterprise software systems.

---

## **Why Java is Used**

1. Used to build Android apps, desktop and web apps, enterprise backend systems, and cloud based software.
2. Has popular frameworks like Spring and Hibernate which makes it powerful for enterprise applications.
3. Supports object-oriented programming for clean, maintainable and reusable code.
4. Platform independent due to Java Virtual Machine (JVM).
5. Strong community support and extensive libraries.

---

## **Key Concepts**

1. Java is a platform-independent language, which means code written in java can run on any device that supports the Java Virtual Machine (JVM).
2. Java follows the principle Write Once, Run Anywhere (WORA) meaning once program is compiled into bytecode by Java compiler, it can run on any system that has a JVM, regardless of the underlying operating system.
3. Java is both compiled and interpreted. Code is compiled into the bytecode using the Java compiler and Bytecode is executed by the JVM.

---

# **Features Of java**

---

## **1. Platform independent**

Java programs can run on any operating systems without modification.

### **How it works:**

* Java source code (.java) is compiled into the bytecode (.class).
* Bytecode is exceuted by the JVM.
* JVM acts as bridge between code and OS.

### **Example:**

A java program compiled on Windows can run on Linux or macOS without recompilation.

---

## **2. Object-Oriented**

Java is based on the concept of classes and objects.

### **Why it matters:**

* Promote code reusability (Inheritance).
* Improves code organization (Encapsulation).
* Supports flexibility (Polymorphism).
* Hides implementation details and shows only essential features (Abstraction).

### **Impact:**

Helps in building modular and maintainable applications.

---

## **3. Robust**

Java is designed to be reliable and reduce errors during execution.

### **Why:**

* Strong exception handling mechanism.
* Automatic memory management (Garbage Collection).
* Compile-time and runtime checks.

### **Impact:**

Reduces chances of crashes and improves stability.

---

## **4. Secure**

Java provides a secure execution environment.

### **Why:**

* No direct memory access (no pointers like C/C++).
* Bytecode verification before execution.
* Security manager and sandbox environment.

### **Impact:**

Prevents unauthorized access and protect system resources.

---

## **5. Portable**

Java programs can be easily transferred accross different environments.

### **Why:**

* Platform-independent bytecode
* No dependency on hardware-specific features

### **Impact:**

Same program behaves consistently across systems.
